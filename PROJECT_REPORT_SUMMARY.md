# ESP32-P4 智能电子鼻边缘感知系统 — 项目总结汇报文档

## 1. 项目简介 (Project Introduction)
本项目是一款基于 **ESP32-P4** 高性能主控芯片和 **ESP32-C6** 协处理器的**智能电子鼻（E-Nose）边缘感知与交互系统**。系统通过自主研发的双芯片高速通信架构（ESP-Hosted over SDIO），实现了环境气体高精度感知、大模型智能分析、实时图像识别与人性化语音交互的多模态融合。本系统可广泛应用于食品新鲜度检测、智能仓储温湿气监测、果蔬熟化度评定及环境空气质量监测等物联网边缘计算场景。

---

## 2. 核心硬件平台 (Hardware Configuration)
| 模块名称 | 芯片/器件型号 | 关键技术规格与作用 |
|:---|:---|:---|
| **主控核心** | **ESP32-P4** | 双核 240MHz RISC-V，16MB Flash，8MB PSRAM。负责主应用逻辑、摄像头流处理、音频 Opus 解码、LVGL UI 高速渲染及本地数据集成。 |
| **无线通信协处理器**| **ESP32-C6** | 负责 Wi-Fi 6 (2.4GHz) 及 BLE 5.0 协议栈运行，通过 SDIO 接口为 P4 提供高速无线网络和蓝牙连接。 |
| **显示屏幕** | **7寸 MIPI-DSI 屏**| 分辨率 **1024x600**，支持多点电容式触摸，基于 LVGL v9 运行，提供丝滑的触屏交互。 |
| **摄像头模块** | **SC2336 MIPI-CSI**| 200万像素，1280x720@30FPS 硬件视频采集，提供低延迟扫码与网页端监控图像源。 |
| **音频解码芯片 (Codec)**| **ES8311** | 配合板载麦克风和双扬声器（带 GPIO 53 功放使能），支持录音与高质量语音回放。 |
| **环境传感器** | **BME680** | I2C 接口，高精度监测环境温度、湿度及大气压强（0x77 物理地址）。 |
| **分布式气体检测节点**| **ESP32-S3 (多节点)** | 搭载 VOC、CO2、Ethylene（乙烯）等高精度气体传感器，通过 BLE 发送实时监测数据。 |

## 3. 功能模块详解 (Core System Features)

### 3.1. 多节点 BLE 气体传感器网络 (Distributed BLE Sensor Network)
- **蓝牙网关技术**：主控 P4 调用 C6 的 NimBLE 协议栈，作为 BLE Central 主动扫描并连接多个 S3 传感器子节点（最多支持 4 节点并发连接，支持断线自动重连）。
- **轻量级协议**：节点数据以 JSON 格式无线广播或订阅传输：`{"node":"S3_A","voc":1.2,"co2":520,"eth":2.3}`。
- **动态适配**：主控接收到 BLE Notify 后，通过 cJSON 解析，自动更新对应屏幕看板及 Web 端指标。

### 3.2. 小智 AI 语音交互系统 (XiaoZhi AI Voice Assistant)
- **全双工对话**：集成小智 AI 语音服务，基于 WebSocket V2 协议提供流畅的在线对话体验。
- **PTT 录音机制**：设计 Push-To-Talk（按住说话，松开发送）按键机制，完美适应嘈杂工作环境。
- **Opus 编解码**：采集音频后，使用 Opus 编解码器（16kHz单声道，20ms帧，12kbps）进行高效压缩传输，大大节省带宽并保障低延迟。
- **多任务音频管道**：在 FreeRTOS 中分离麦克风采集、Opus 解码、扬声器播放任务，配合 RingBuffer 缓存，防止音频断音或阻塞。

### 3.3. 火山引擎“豆包”AI 气体分析网关 (Volcano Doubao AI Gateway)
- **大模型赋能**：系统内置 HTTPS 智能网关。在点击“AI分析”时，自动打包本地 BME680 数据及所有 BLE 节点的 VOC、CO2、乙烯等多维气体指标。
- **自定义 Prompt 专家库**：结合特定提示词（如新鲜度评估标准、变质预警机制）调用火山引擎“豆包（doubao-seed-1.6）”大模型 API。
- **结果可视化**：将云端大模型生成的专业诊断报告（包含新鲜度评级、食用建议、存储改进等）以 CJK 中文字库平滑呈现在 7 寸触控屏的专属弹窗中。

### 3.4. LVGL v9 触屏人机交互界面 (LVGL v9 Graphic Interface)
设计了精美、动感的三个独立物理屏幕：
1. **启动屏 (Startup Screen)**：支持实时 Wi-Fi 扫描，弹出密码输入及虚拟键盘模态框，直观展示联网状态并一键进入系统。
2. **主看板 (Main Dashboard)**：卡片式布局，实时呈现本地 BME680 数据及分布式 S3_A / S3_B 节点气体浓度；集成 AI 语音球状态波形，实时反馈小智助手（Ready/Listening/Thinking/Speaking）的状态；配有音量与亮度微调拉条。
3. **摄像头扫码屏 (QR Scanner Screen)**：显示 MIPI-CSI 摄像头实时流，并提供绿色对焦框和扫码节点绑定反馈。
4. **控制中心 (Control Center)**：支持下拉手势弹出，可快捷配置连接 Wi-Fi，并利用滑块无缝控制设备音量与亮度。

### 3.5. 摄像头 V4L2 采集与扫码绑定 (SC2336 Camera & QR Decoder)
- **V4L2 高速管道**：利用 ESP-Video 框架驱动 SC2336，建立 PSRAM 帧队列（4 缓冲，USERPTR 模式），实现 1280x720@30FPS 视频数据采集。
- **QR 扫码节点绑定**：将取样帧缩放为 200x150 灰度图像，载入 `quirc` 解码器。若识别到指定节点二维码（如 S3A），系统自动绑定节点，简化工业现场传感器配对流程。

### 3.6. 远程嵌入式 Web 仪表盘 (Embedded Web Dashboard)
- **HTTP 远程监控**：板载嵌入式 Web 服务器。局域网用户只需在浏览器输入设备 IP 即可访问响应式（PC/移动端自适应）深色主题数据大屏。
- **零等待 JPEG 压缩流**：在视频采集线程中，利用 ESP32-P4 的 **硬件 JPEG 编码器** 自动将 720P 帧缩放并转换为 256x144 JPEG 图片（Q=50，4:2:0采样），Web 前端采用无闪烁图片轮替，提供 ~5fps 的超低 CPU 占用网页监控画面。

## 4. 系统资源优化与稳定性保障 (System Optimization)
1. **PSRAM 深度利用**：LVGL 渲染引擎、摄像头双显示缓冲区、小智音频环形缓冲区（Opus RingBuffer 64KB，PCM RingBuffer 128KB）均通过 `heap_caps_malloc` 分配在板载 8MB PSRAM 中，保障 120KB 珍贵内部 SRAM 不会溢出。
2. **多核亲和性与吞吐优化**：将高性能消耗任务（如 Opus 音频处理、LVGL UI 渲染）绑定在 Core 1，将数据采集任务（如 V4L2 摄像头读取）绑定在 Core 0，最大限度发挥双核 RISC-V 架构性能。
3. **中文字体 Fallback 机制**：配置 `s_font_cjk_fallback`，以 `lv_font_montserrat_14` 为主，挂接中文字库（如思源黑体二进制字库 `font.bin`），完美解决中文长文本显示方块字/乱码问题，节省大量固件空间。
4. **无锁数据传递**：采用双缓冲加自旋锁保护 Web 端 JPEG 数据，在数据安全的同时避免了线程阻塞，保障 UI 交互任务响应时间在 30ms 以下。

## 5. 构建与部署说明 (Build & Flash)
### 5.1. 编译依赖
- **编译工具链**：ESP-IDF v5.5.2 或以上版本
- **外部组件**：LVGL v9, NimBLE (BLE), quirc (二维码), esp_jpeg_enc (硬件 JPEG 编码器)

### 5.2. 编译烧录指令
```bash
# 1. 设定目标芯片为 ESP32-P4
idf.py set-target esp32p4

# 2. 编译项目
idf.py build

# 3. 烧录固件到开发板 (将 COMXX 替换为实际串口号)
idf.py -p COMXX flash

# 4. 启动串口监控查看运行日志
idf.py monitor
```
