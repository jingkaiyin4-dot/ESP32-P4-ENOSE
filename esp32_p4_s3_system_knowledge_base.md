# ESP32-P4 与 ESP32-S3 电子鼻智能网关项目外设与功能知识库

本知识库专为软硬件开发与系统联调提供核心技术指导，内容高度聚焦于本项目的**测量化学分子原理**、**底层硬件外设工作机制**以及**系统交互控制功能**，精简了宽泛的系统架构理论。

## 二、 气体分子表征与食材新鲜度原理

电子鼻系统通过捕获食材在变质、发酵或成熟过程中释放的特定气味分子，来实现对新鲜度的数字化表征。系统监测的 5 种核心气体分子及其表征意义如下：

### 1. VOCs (挥发性有机化合物分子，如乙醇、乙烯)
- **代表气体**：乙醇（酒精）、乙烯（植物催熟激素）、硫化氢及硫醇类（肉类蛋白质腐败释放的恶臭分子）。
- **表征作用**：食材（尤其是水果和肉类）在成熟、发酵或腐败变质时，微生物会将糖类或蛋白质分解，产生大量的乙醇与硫化分子。VOCs 浓度的陡增是食材发生变质和发酵最核心、最灵敏的化学证据。

### 2. Odor (异味/特征气味混合分子)
- **代表气体**：氨气、三甲胺（鱼类及海鲜腐败的特有腥臭分子）、低级脂肪酸等。
- **表征作用**：综合反映空气中具有强烈人体感官特征的混合有机气味分子的 ppm 相对总浓度，直接对应人眼/人鼻对食材是否变质的第一直观判断。

### 3. HCHO (甲醛/甲醛类蚁醛分子)
- **代表气体**：低级醛类气体、甲醛分子。
- **表征作用**：某些富含蛋白质的食材被特定细菌分解时，会伴随低级胺类与低级醛类分子的挥发。甲醛传感器对这类蚁醛分子具有极高的端侧捕捉精度，常用作肉类初期变质的敏感特征量。

### 4. CO (一氧化碳/碳氧化合物分子)
- **代表气体**：微量 CO 分子。
- **表征作用**：有机物在受限空间内发生缓慢不完全氧化或复杂厌氧发酵时，常伴有微量 CO 气体的释放。作为特征辅助维度，用于提高多传感器数据融合算法的抗干扰能力。

### 5. CO2 (二氧化碳分子)
- **代表气体**：CO2 气体。
- **表征作用**：二氧化碳是好氧微生物（如霉菌、细菌）进行有氧呼吸代谢的直接产物。密封或受限空间内 CO2 浓度的迅速攀升，是微生物大量滋生、食材被微生物污染的直接物理指标。

## 二、 核心外设 (Peripherals) 硬件机制与引脚配置

项目中各芯片连接的物理外设涵盖模拟采样、串行通信、高阻抗 SPI 及大电流继电器驱动，其工作参数与底层驱动细节如下：

### 1. 模拟采样 ADC 外设 (DFRobot_ADS1115)
- **硬件作用**：16 位超高精度 I2C 模数转换器，用于采集 4 路半导体气体传感器（分别针对 VOC, HCHO, CO, Odor）的模拟输出电压。
- **底层参数配置**：
  - **增益配置**：`eGAIN_TWOTHIRDS`（量程范围为 `±6.144V`，单阻抗输入，可完美容纳传感器在 0~5V 范围内的电压起伏）。
  - **采样率配置**：`eRATE_128` (128 SPS，单次触发转换以降低功耗和通道串扰)。
  - **转换模式**：`eMODE_SINGLE` (单次触发转换模式)。
- **I2C 物理引脚**：`SDA (GPIO 18)`、`SCL (GPIO 17)`。

### 2. CO2 NDIR 检测外设 (MH-Z19C)
- **硬件作用**：非色散红外 (NDIR) 二氧化碳传感器，检测范围为 400 ~ 5000 ppm。
- **底层通信机制**：
  - **串行外设**：通过 ESP32-S3 的 `HardwareSerial (UART1)` 进行点对点数字通信。
  - **波特率配置**：`9600` bps，数据位 `8`，停止位 `1`，无校验。
  - **恢复与容错**：软件中加入了 `mhz19FailCount` 连续读取失败计数器，一旦连续 5 次读取失败，系统会自动清除健康状态位，并触发 I2C/串口的硬件级 `reinitSensors` 重初始化以防死锁。
- **物理引脚**：`RX (GPIO 16)`、`TX (GPIO 15)`。

### 3. 环境补偿感知外设 (BME680)
- **硬件作用**：集成式温度、湿度、气压及气体阻抗传感器。采集到的温度与相对湿度直接作为 S3 侧神经网络温湿度补偿器 (NN MLP) 的核心输入特征。
- **底层工作配置**：
  - **物理总线**：HSPI 硬件 SPI 外设。
  - **采集模式**：强制单次测量模式 `BME68X_FORCED_MODE`。
  - **过采样配置**：温度过采样 `BME68X_OS_8X`，湿度过采样 `BME68X_OS_2X`，不启用压力采集以缩短测量耗时。
  - **数字滤波**：配置 `BME68X_FILTER_SIZE_3` 硬件低通滤波器，过滤气流扰动带来的读数突变。
- **SPI 物理引脚**：`MOSI (GPIO 10)`、`MISO (GPIO 11)`、`SCK (GPIO 12)`、`CS (GPIO 5)`。

### 4. 紫外消杀继电器外设 (Relay Control)
- **硬件作用**：大电流物理继电器，控制高压紫外线杀菌保鲜灯的物理供电。
- **底层驱动机制**：
  - 采用标准 GPIO 输出外设直驱继电器光耦输入端。
  - **防电磁干扰 (EMI)**：继电器吸合瞬时会产生反向感应电动势（电感性浪涌）。底层硬件上加装了续流二极管保护，软件上在继电器吸合及释放前后加装了 `delay(10)` 的 CPU 避峰避噪延时，防止高频射频（ESP-NOW/Wi-Fi）发生瞬时死机或复位。
- **物理引脚**：`Relay PIN (GPIO 14)`。

### 5. DPI LCD 高清屏显示外设 (EK79007)
- **硬件作用**：7寸 DPI 高清触控屏，1024x600 物理分辨率，DSI 物理层直驱。
- **底层加速配置**：
  - 开启 **DMA2D 硬件搬运拷贝** 引擎，用于多帧缓冲区（3个屏缓冲区）的零 CPU 拷贝合成。
  - 配置 `CONFIG_LV_USE_PPA=y`，使用 **PPA (Pixel Processing Accelerator) 硬件图像像素处理加速器**，直接进行零拷贝的摄像头 RGB565 到灰度 GRAY8 的硬件降采样与色彩转换，完全释放了 CPU 算力。

### 6. I2S 喇叭与 Codec 音频外设
- **硬件作用**：连接外部 I2S 音频解码器芯片，驱动 8Ω 喇叭，用于输出流畅的小智 AI TTS 语音反馈。
- **底层加速配置**：
  - **DMA 安全静态化**：播放临时混音区 `stereo_chunk` 被限制在内部 SRAM 数据段（通过 `static int16_t stereo_chunk[512]`），确保 I2S DMA 硬件能够以零等待的物理速率直接写入 Codec，彻底抹平了声音卡顿。
  - **功耗控制**：功放使能脚为 `GPIO 53`。

## 三、 系统核心控制功能与指令集

整个系统通过三级联动，实现了一套包含“端侧模型扫描”、“多级消杀控制”及“前导滤噪心跳”的完整业务功能：

### 1. 食材新鲜度模型动态扫描与加载功能
S3 接收端具备从物理 SD 卡动态扫描和热加载 TensorFlow Lite 推理模型的能力：
- **扫描功能 (`scanModels`)**：系统在初始化或接收到 `model rescan` 串口命令时，会利用 SD 库扫描 SD 卡根目录，筛选所有后缀为 `.tflite` 的文件，并将模型信息加载到结构体数组中。
- **热加载功能 (`loadModelByIndex(N)`)**：系统在开机或接收到 `model load N` 命令时，会调用 `loadModelByIndex(N)` 动态读取对应的 `.tflite` 文件，重新初始化微型 Tensor 推理池并分配 Tensor 空间，从而在**不重新烧录固件的前提下，直接热更换针对不同食材（如水果类、肉类、海鲜类）的新鲜度分类模型**。

### 2. 多级紫外消杀控制与指令集
消杀控制支持 P4 屏幕手动直接控制、C6 串口调试控制与 S3 本地根据新鲜度阈值自动判定控制。C6 接收并转发的指令细节如下：

#### 💬 紫外消杀功能指令集表
| 功能指令 | 对应底层执行动作 (C6 自动转换下划线并经 ESP-NOW 送达 S3) | 业务功能详述 |
| :--- | :--- | :--- |
| **`uv on`** | 切换 Relay PIN 为 high，开启紫外灯；启动 `uvStartTime` 计时器。 | **手动强制开启**：无条件点亮紫外杀菌灯，进入定时消杀倒计时。 |
| **`uv off`** | 切换 Relay PIN 为 low，关闭紫外灯；清除消杀计时器。 | **手动强制关闭**：立即熄灭紫外杀菌灯，复位所有消杀状态。 |
| **`uv auto on`**| 设置 `uvAutoMode = true`，使能自动判定。 | **开启自动智能消杀**：交由 S3 本地模型判定。当新鲜度评分 `fr` 低于设定阈值，且类别非空气基准时，自动开启紫外杀菌。 |
| **`uv auto off`**| 设置 `uvAutoMode = false`，去使能自动判定。 | **关闭自动智能消杀**：退回手动控制状态。 |
| **`uv dur N`** | 修改 `uvOnDuration = N * 1000`（毫秒）。 | **设定消杀持续时间**：配置单次消杀的最长点亮时间（单位：秒，如设定 `uv dur 30` 代表单次开启 30 秒后自动熄灭）。 |
| **`uv thresh N`**| 修改自动开启消杀的新鲜度阈值。 | **设定消杀敏感度**：例如设定 `uv thresh 70`，则当新鲜度 `fr` 跌破 70 分时自动点亮紫外灯进行杀菌保鲜。 |

### 3. P4-C6 串口前导脏字符智能滤噪心跳功能
为确保 P4 网关能高抗噪、绝对稳定地显示心跳和数据，系统设计了以下容错功能：
- **心跳机制**：C6 网桥每隔 30 秒自动通过 UART1 向 P4发送心跳 JSON。如果 ESP-NOW 数据链正常，发送 `{"bridge":"heartbeat","ble":"up","age":N}`（`age` 为距上一次收到传感器数据的时间，秒）；若传感器掉线，自动呈报 `{"bridge":"heartbeat","ble":"down"}`。
- **前导滤噪功能**：当 P4 串口由于硬件拔插或瞬时干扰产生乱码时，P4 端的 `parse_and_notify` 会自动在行缓冲区中检索首个左大括号 `{`。以此作为真正的 JSON 起始指针，直接过滤掉前面的所有脏字节，确保心跳监测和传感器数据呈现 100% 不会因为数据帧被噪声污染而导致系统异常或崩溃。
